## 2. Directory Analysis

| Directory | Responsibility & Purpose | Key Dependencies | Importance |
| :--- | :--- | :--- | :--- |
| **`src/app/actions`** | Exposes database mutation pipelines (Server Actions). | `drizzle-orm`, `next/cache` | **Critical**: Handles all data operations without open API routes. |
| **`src/app/admin`** | Serves as the operator control center interface. | `verifyAuthSession` helper | **High**: Coordinates administrative modifications to the database. |
| **`src/app/blog`** | Displays technical markdown posts to visitors. | `drizzle-orm` queries | **Medium**: Main content reader area. |
| **`src/components`** | Visual presentation modules for UI pages. | `framer-motion`, `lenis` | **High**: Defines interaction aesthetics and design systems. |
| **`src/db`** | Controls Drizzle compiler configurations & schemas. | `@libsql/client` | **Critical**: Ensures structure validity in the SQLite local db. |
| **`src/lib`** | Utility systems (Authentication, parsing, math). | `bcryptjs`, `jose`, `clsx` | **Critical**: Core helper functions (auth checks, reading speed). |
| **`backup_static`** | Legacy draft archive storage. | None | **None**: Dead storage, can be removed. |

---

## 3. Structural Anomalies

### 3.1 Unused / Dead Folders
- **`backup_static/`**: Contains `ai-portfolio.html` which is a static legacy design code placeholder. Unused by Next.js compiler loops. Can be archived safely outside the runtime path.

### 3.2 Duplicate Folders
- **None**: Structural layouts are highly isolated and clear of direct subfolder overlaps.

### 3.3 Missing Folders
- **`src/db/migrations/`**: Stated as the output folder target in [drizzle.config.ts](file:///d:/portfolio/drizzle.config.ts), but does not exist in the active tree. This implies schemas are currently being synchronized directly with `drizzle-kit push:sqlite` instead of managed migrations files.
- **`tests/`** or **`src/__tests__/`**: No directory exists for unit testing layouts or action functions, indicating a lack of coverage tooling.
